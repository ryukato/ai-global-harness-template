# Project Tech Stack Skill

## Purpose

Apply project-specific technical constraints consistently.

## Required References

Read these files when relevant:

- `docs/architecture/tech-stack.md`
- `docs/architecture/coding-rules.md`
- `docs/architecture/testing-strategy.md`
- `docs/architecture/build-and-run.md`
- `docs/architecture/dependency-policy.md`

## Instructions

1. Identify the affected layer or module.
2. Follow existing framework and coding style.
3. Avoid adding new libraries unless clearly justified.
4. Use the existing test framework.
5. Use existing build commands where available.
6. Preserve compatibility with existing runtime and deployment assumptions.

## Output Expectations

Mention any technology assumption that affected the implementation.
