# Release Checklist Skill

## Purpose

Validate readiness before release or handoff.

## Checklist

- Acceptance criteria satisfied
- Tests executed
- Migration impact checked
- Configuration impact checked
- Backward compatibility considered
- Observability/logging impact considered
- Rollback or recovery approach considered
- Documentation updated if needed

```text
Release Readiness:
- Ready / Not Ready

Evidence:
- ...

Risks:
- ...
```
