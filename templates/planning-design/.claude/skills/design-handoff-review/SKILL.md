# Design Handoff Review

Use this skill when a screen, flow, prototype, or design note is about to move to engineering.

## Checklist

- Core flow is covered.
- Empty, loading, error, and permission states are present.
- Responsive behavior is described.
- Copy, labels, validation messages, and localization notes are available.
- Data dependencies and API assumptions are clear.
- Accessibility expectations are documented.
- Analytics or event tracking needs are documented when relevant.

## Output

Return:

- Summary
- Ready / Not Ready
- Missing States
- Engineering Questions
- Suggested Jira Follow-ups
