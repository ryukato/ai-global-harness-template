# Requirement Review

Use this skill before a requirement is converted into implementation work.

## Checklist

- User problem and outcome are clear.
- In-scope and out-of-scope items are explicit.
- Acceptance criteria are testable.
- Dependencies and assumptions are visible.
- Data, permissions, and operational constraints are stated.
- Open questions are separated from confirmed requirements.

## Output

Return:

- Summary
- Must Fix
- Should Fix
- Open Questions
- Suggested Acceptance Criteria
