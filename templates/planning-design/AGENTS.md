# Planning / Design Agent Guide

This workspace is optimized for product planning and design collaboration.

## Operating Rules

- Start from `docs/planning/*`, `docs/design/*`, and `docs/operations/*`.
- Keep outputs concise, traceable, and ready for review by product, design, and engineering.
- Do not assume this repository contains application code.
- Do not write to Jira, Figma, Confluence, GitHub, or other external systems without explicit user approval for the exact change.
- Never store API keys, Personal Access Tokens, cookies, or session values in the repository.

## Preferred Outputs

- Product brief updates
- Requirement review notes
- Acceptance criteria
- Jira issue drafts
- Figma requirement review notes
- Design handoff checklists
