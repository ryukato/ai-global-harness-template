# Figma Review Checklist

Use this checklist before moving a design into requirement or engineering review.

- Frames match the documented requirements.
- Primary flow is complete.
- Empty, loading, error, disabled, and permission states are represented when relevant.
- Form validation messages are visible.
- Responsive behavior is documented or represented.
- Copy is final enough for implementation.
- Data sources and dynamic values are identified.
- Accessibility expectations are documented.
- Open design decisions are clearly marked.
