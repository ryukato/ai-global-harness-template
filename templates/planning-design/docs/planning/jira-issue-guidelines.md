# Jira Issue Guidelines

## Issue Quality

A ready Jira issue should include:

- Clear summary
- Context and goal
- Requirements
- Acceptance criteria
- Dependencies
- Source links
- Owner or requesting team
- Open questions

## Splitting Guidance

Split issues when:

- One issue covers multiple user outcomes.
- Frontend and backend work can be reviewed independently.
- Design, copy, data, or operational work has separate owners.
- The acceptance criteria cannot be tested as one coherent unit.

## External Write Rule

Agents may draft Jira issues, but must not create or update Jira without explicit approval for the exact content.
